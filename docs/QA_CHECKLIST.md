# FIXEL QA 체크리스트

## 1. 이미지 업로드

### 기능 테스트
- [ ] 클릭으로 파일 선택 후 업로드
- [ ] 드래그앤드롭으로 업로드
- [ ] 업로드 후 에디터 페이지(/editor)로 자동 이동
- [ ] 업로드된 이미지가 캔버스에 정상 표시

### 엣지 케이스
- [ ] JPG 파일 업로드 성공
- [ ] PNG 파일 업로드 성공
- [ ] WebP 파일 업로드 성공
- [ ] 10MB 초과 파일 → 에러 메시지 표시
- [ ] 지원하지 않는 형식(GIF, SVG, PDF) → 에러 메시지 표시
- [ ] 4096px 초과 이미지 → 에러 메시지 표시
- [ ] 10px 미만 이미지 → 에러 메시지 표시

- [ ] "배경 제거 실행" 버튼 → 로딩 표시 → 결과 반영
- [ ] 결과 이미지에 투명 배경 (체크보드 표시)
- [ ] 엣지 리파인 자동 적용 체크박스 동작

- [ ] 처리 중 다른 도구 클릭 방지 (isProcessing)
- [ ] 인물/상품/복잡한 배경 등 다양한 피사체 테스트

## 3. 톤 보정

### 기능 테스트
- [ ] 밝기 슬라이더 -100 ~ +100 범위 조절
- [ ] 대비 슬라이더 -100 ~ +100 범위 조절
- [ ] 채도 슬라이더 -100 ~ +100 범위 조절
- [ ] 색온도 슬라이더 -100 ~ +100 범위 조절
- [ ] "적용" 버튼 → 이미지에 필터 반영
- [ ] "초기화" 버튼 → 슬라이더 0으로 복귀
- [ ] Toast 알림: "톤 보정이 적용되었습니다." 표시

### 엣지 케이스
- [ ] 모든 슬라이더 0일 때 적용 → 이미지 변화 없음
- [ ] 극단값(-100, +100) 적용 → 깨지지 않음

## 7. 배경 교체

### 기능 테스트
- [ ] 투명 배경 선택
- [ ] 단색 배경: 컬러 피커로 색상 선택 + 프리셋 색상
- [ ] 그라데이션: 시작/끝 색 + 6방향(→←↓↑↘↙)
- [ ] 이미지 배경: 8종 기본 이미지 선택
- [ ] 이미지 배경: 사용자 업로드
- [ ] 커머스 템플릿: 스마트스토어/쿠팡/당근/11번가 선택
- [ ] "배경 적용" → 합성 결과 반영

### 엣지 케이스
- [ ] 누끼 없이(투명 영역 없이) 배경 교체 → 원본 위에 겹침
- [ ] 가로/세로 비율 다른 배경 이미지 → cover 방식으로 채움

## 8. Undo/Redo

### 기능 테스트
- [ ] "← 취소" 버튼 → 이전 상태로 복원
- [ ] "다시 →" 버튼 → 취소한 상태 재적용
- [ ] Ctrl+Z → Undo 동작
- [ ] Ctrl+Shift+Z → Redo 동작
- [ ] Ctrl+Y → Redo 동작
- [ ] 히스토리 카운트 표시 (예: "3 / 5")
- [ ] 최대 20단계 히스토리 유지

### 엣지 케이스
- [ ] 히스토리 0일 때 Undo 비활성
- [ ] 히스토리 끝일 때 Redo 비활성
- [ ] 20단계 초과 시 오래된 히스토리 자동 삭제
